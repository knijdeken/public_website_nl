E-mail templates

Commence and MemoTools can do a marvelous job with e-mail templates. That is, we hope, no news. But when we implemented our new house style that was a good moment to restructure our e-mail templates. We had templates that were created by cut and paste and thus not very smart.

The new templates are based upon three parts:

  • a generic header;
  • a body;
  • a generic footer.

Special MemoCodes allow the body to include the header and the footer and to create the final template.

Header and footer blocks do not refer to a category. In the header you typically define some generic MemoCodes and for example the HTML header and body tags.

The footer contains the signature and some closing tags. And the smart part is the body, for example:

(%(-INCLUDE-)#HEADER%)
<p>Salutation</p>
(%(-INCLUDE-)#FOOTER%)

The result of all this work is that we have only two headers and two footers (one English and one Dutch) for all the e-mail templates and that the e-mail template bodies are much smaller. We have uploaded an example of our e-mail templates to TechNews.

MemoTools 4.0

With the release of Commence RM 4.0, there is also a new release of the MemoTools. Here is an overview of some of the changes:

  • one manual with all the tools (the MemoDLL manual is still a separate manual) instead of many small ones;
  • all tools now comply to the new Memocom style;
  • all tools are installed in the Memocom directory;
  • the configuration files of the tools are now called toolname.cfg;
  • all tools are updated to Commence RM 4.0;
  • new MemoLink tool, allowing you to connect files to Commence objects (with righthand mouse click);
  • MemoExport tool added. MemoExport is a tool that combines some older tools and can be used to export data from Commence in many formats and with many conversions. Even custom scripting is supported.


New/changed features in existing tools:

  • MemoMonitor can process the new workgroup sync settings;
  • MemoMonitor does no longer have an embedded webserver since this would cause conflicts with other components that use a standard Microsoft or Apache webserver;
  • MemoDoc uses a .css file to create the output;
  • MemoDLL has a number of new and changed features and some bugfixes.

Script examples

When you install the MemoTools development version, we will create a directory C:\Program Files\Memocom\Examples. In this example directory, you will find example scripts that use the majority of the MemoDLL calls. You can just copy and paste the code and not reinvent.
