Commence allows you to add scripting to detail forms (DTF). This can be as simple as creating a script to validate a field or as complex as scripting a complete workflow. This edition of TechNews will explain you the basics on how to do this.

Step 1 Use ScriptPad

Using the DTF script functions under 'Customize->Database->Detail form->edit->scripting' you can check out, edit and check in a script. This method is very time consuming and is not very practical. A more efficient way is to use a script editor called Scriptpad3 (available on our Technews site under /Tools).

Using Scriptpad you can view and edit the script immediately by selecting the related database, category and form.

Step 2 Reuse

There are three standard templates available for DTF:

• A simple one, with comments and the main event handlers;
• A simple one, with the logging system;
• An advanced with all possible event handlers and logging;

Note: the logsystem is an implementation of a system that maintains the differences in fields in a record and then stores these in the logfield. This way the user can see when and by whom certain data has been changed.

Step 3 Start playing

The best way to learn programming Commence is by doing it! So create a test database (or download the database Templates.xfr) and start scripting.

Step 4 Example add a BSN numbercheck

We will use template 2 and add a BSNnumber check to it.

• First add a field BSNnumber to the category democat. This field is a text field and is max 10 characters long.

• We want to do a check when we leave the field, so we go to the related event handler...

• An extra case statement needs to be added which the script will jump to when we leave the BSNnumber field.
• The function that tests the BSNnumber with an 11 proof is part of MemoDLL, so only one call will do the job.
• In case of a wrong number, we will prompt the user to reenter the number or to empty the field (cMsg1).
