MemoDLL

What is MemoDLL?

MemoDLL is a core module of the CRM application that you build with Commence. Almost all screens will use functions collected in MemoDLL. But also standalone scripts in for example VBscript, PHP or Ruby can use functions from MemoDLL. All the MemoTools use functions of MemoDLL too.

What can you do with MemoDLL?

MemoDLL contains a huge collection of functions. Some examples:

• Functions for merging Word and e-mail templates;
• A function to send e-mail (this is a complete e-mail solution);
• Functions to translate text;
• Functions to check what data changed in a form;
• Functions like zipcode check, kvk check, BSN check, account number etc. etc.

Functions that have not been covered (until today) are the log functions. In today’s world it becomes more and more important to keep track of what software is actually doing. Important decisions made by software must be logged for audit purposes or for easy debugging.

MemoDLL already uses log functions from the very first release but in the 4.0 release we have made the logfunctions available to the outside world too. Here is an overview of the log functions:

Function to log a message in the active.log of MemoDLL:

vCcode = oMemoTools.LogMsg( pMsg )

Function with trace level support to log a message in the active.log of MemoDLL, the message will only be logged in the active.log of MemoDLL when the tracelevel set for MemoDLL (stored in the registry) is equal to or greater than the tracelevel set in the call (tracelevel is between 0 and 99 )

vCcode = oMemoTools.LogMsg2( pMsg, pTraceLevel )

Function to log a message in a file, but with the same layout as the active.log of MemoDLL (the file is always stored in the same directory as the active.log; pFilename must just be a filename, so not a path).

vCcode = oMemoTools.LogMsg3( pFilename, pMsg )

Please note: the first two functions have an active.log as a circular file, i.e. at some point, the first part of the file is truncated. The last function will make a logfile that keeps on growing.

E-mail templates

In the previous TechNews we presented a new way to create e-mail templates using a header and a footer template, both are included in the final template. The result is that all templates become easier to maintain. When you use this method you probably need some additional tools that we describe here.

We have three new scripts available:

ExportEmailTemplates.vbs, this script exports all templates to a directory.
ImportEmailTemplates.vbs, this script imports all templates from a directory.

The e-mail templates are stored as text files. To be able to test them we have a third script:

TestEmailTemplates.vbs, this script creates real HTML files that you can view in your browser.

So when you are going to move your e-mail templates to the new structure, here are the steps to take:

• Create a header template;
• Create a footer template;
• Create a new e-mail template that uses header and footer;
• Adjust all three until you are satisfied;
• Use the ExportEmailTemplates.vbs script to export all the templates;
• Using a text editor (for example Notepad++) change all the other templates;
• Use the TestEmailTemplates.vbs script to generate .html files;
• Check all generated html files in a browser;
• Use the ImportEmailTemplates.vbs script to import all the e-mail templates;
• Test all e-mail templates using Commence.

Large text fields

Commence supports many field types and within some field types Commence also has a number of options. When you use the text field type you have the option to choose a small (80 chars or less) or large (more than 80 chars) textfield. When you select "large", Commence will create an extra file in the database to store this large text field. So for every large text field, Commence will open an extra file.

At this moment the number of open files inside Commence is set to 80, so there is a limit to the amount of large text fields that you can set in a category. When you have problems in an agent on the category or you can’t read or store data in a field, please check the number of large text fields in the category.

Script examples

We have a new directory one the ftpsite with example scripts (technews\scripts). All scripts are in VBS and have the extension .txt. When you like to use a script, download it, change it the way you like and rename the extension to .vbs . Finally run and test it. We hope this will allow you to use MemoDLL functions more and more and thus reduce the amount of script you need in Commence itself.
